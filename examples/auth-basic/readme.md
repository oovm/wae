# WAE Authentication 基础示例

本示例展示了 WAE 框架中认证与授权的基本使用方法。

## 功能说明

本示例演示了基本的认证与授权功能：

- **用户登录**: 用户名/密码认证
- **Token 管理**: Token 生成与验证
- **角色控制**: 基于角色的访问控制 (RBAC)
- **权限检查**: 细粒度的权限验证

## 运行方式

```bash
cargo run --package auth-basic
```

## 测试账号

- **管理员账号**
  - 用户名: `admin`
  - 密码: `admin123`
  - 角色: `admin`, `user`
  - 权限: `read`, `write`, `delete`

- **普通用户账号**
  - 用户名: `user`
  - 密码: `user123`
  - 角色: `user`
  - 权限: `read`

## 示例内容

1. **初始化认证服务**
   - 创建用户和角色数据

2. **管理员登录**
   - 演示用户名/密码认证
   - 生成访问令牌

3. **验证 Token 并获取用户信息**
   - 演示 Token 验证
   - 获取用户详细信息

4. **检查角色和权限**
   - 演示角色检查
   - 演示权限验证

5. **普通用户登录**
   - 演示不同用户的登录流程

6. **验证普通用户 Token**
   - 获取普通用户信息

7. **检查普通用户权限**
   - 演示权限差异

8. **测试错误密码**
   - 演示认证失败场景

9. **测试无效 Token**
   - 演示 Token 验证失败场景

## 注意事项

本示例使用内存数据结构模拟认证服务，实际使用时需要：

1. 配置真实的数据库存储用户信息
2. 使用 JWT 或其他 Token 机制
3. 配置密码哈希和盐值
4. 设置 Token 过期和刷新机制

## 相关模块

- `wae-authentication`: 认证与授权核心模块
- `wae-jwt`: JWT Token 管理（可选）
